# Типы атак на ARP (Address Resolution Protocol)

ARP — это протокол канального уровня, который связывает IP-адреса с MAC-адресами в локальной сети. Поскольку ARP не имеет встроенной аутентификации, он уязвим для различных атак. Вот основные типы:

## 🔹 1. ARP Spoofing / ARP Poisoning
**Самая распространённая атака.**
- Злоумышленник отправляет поддельные ARP-ответы, связывая свой MAC-адрес с IP-адресом другого устройства (например, шлюза по умолчанию).
- **Цель**: перехват трафика (Man-in-the-Middle), прослушка сессий, кража данных.
- **Инструменты**: `arpspoof`, `Ettercap`, `Bettercap`.

## 🔹 2. ARP Cache Poisoning
- Разновидность спуфинга, при которой в ARP-таблицы жертв намеренно вносятся ложные записи.
- Может быть направлена на одного хоста или на всю подсеть.
- Приводит к перенаправлению трафика через атакующего.

## 🔹 3. ARP Flooding / DoS-атака через ARP
- Массовая рассылка поддельных ARP-пакетов для перегрузки коммутаторов или хостов.
- **Цель**: исчерпание ресурсов (переполнение ARP-таблиц), отказ в обслуживании.
- Может вызвать деградацию производительности сети или полный сбой связи.

## 🔹 4. ARP Replay Attack
- Перехват и повторная отправка легитимных ARP-пакетов с задержкой.
- Используется для обхода временных защитных механизмов или дестабилизации сети.

## 🔹 5. Man-in-the-Middle (MitM) через ARP
- Комбинация ARP spoofing + перенаправление трафика.
- Атакующий становится «невидимым посредником» между двумя узлами.
- Позволяет модифицировать, перехватывать или блокировать данные.

## 🔹 6. ARP-based Session Hijacking
- После внедрения в поток трафика злоумышленник может перехватить активные сессии (например, HTTP-куки, токены аутентификации).

---

## 🛡️ Меры защиты

| Метод | Описание |
|-------|----------|
| **Статические ARP-записи** | Ручная привязка IP↔MAC на критичных хостах (масштабируется плохо) |
| **Dynamic ARP Inspection (DAI)** | Функция на коммутаторах Cisco/других вендоров, проверяющая ARP-пакеты через DHCP snooping |
| **ARP-мониторинг** | Инструменты вроде `Arpwatch`, `XArp` для обнаружения аномалий |
| **Шифрование трафика** | TLS/SSL, IPsec — даже при перехвате данные останутся защищёнными |
| **Сегментация сети** | VLAN, микросегментация — ограничивают радиус атаки |
| **Host-based защита** | Брандмауэры, анти-спуфинг правила, kernel-параметры (`arp_ignore`, `arp_announce`) |

---

> ⚠️ **Важно**: Поскольку ARP работает на канальном уровне, атаки эффективны только в пределах одного широковещательного домена (обычно — одна подсеть/L2-сегмент).

Если вы работаете с сетевой безопасностью, рекомендую протестировать уязвимости в изолированной лаборатории и внедрить DAI + мониторинг в продакшене. Нужна помощь с настройкой конкретных средств защиты?
