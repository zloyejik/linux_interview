# Как работает DNS (Domain Name System)

DNS — это «телефонная книга» интернета, которая преобразует удобные для человека доменные имена (например, `google.com`) в IP-адреса (например, `142.250.185.46`), которые используют компьютеры для связи.

## 🔍 Основной процесс разрешения имени

Когда вы вводите адрес в браузере, происходит следующее:

### 1. **Проверка локального кэша**
- Сначала система проверяет кэш браузера и ОС
- Если запись есть и не истекла — используется сразу

### 2. **Запрос к рекурсивному ресолверу**
- Если в кэше нет записи, запрос идёт к DNS-серверу вашего провайдера или публичному (Google `8.8.8.8`, Cloudflare `1.1.1.1`)
- Этот сервер берёт на себя задачу найти ответ

### 3. **Иерархический поиск**
Ресолвер последовательно опрашивает серверы:

```
Корневые серверы (.) 
       ↓
TLD-серверы (.com, .org, .ru) 
       ↓
Авторитетные серверы домена (example.com)
       ↓
Возврат IP-адреса
```

### 4. **Получение и кэширование ответа**
- Найденный IP-адрес возвращается вашему устройству
- Запись сохраняется в кэше на время, указанное в параметре **TTL** (Time To Live)

## 📋 Типы DNS-записей

| Запись | Назначение | Пример |
|--------|-----------|--------|
| **A** | IPv4-адрес домена | `93.184.216.34` |
| **AAAA** | IPv6-адрес | `2606:2800:220:1::` |
| **CNAME** | Псевдоним для другого домена | `www → example.com` |
| **MX** | Почтовый сервер | `mail.example.com` |
| **TXT** | Текстовая информация (проверки, SPF) | `"v=spf1 include:_spf.google.com"` |
| **NS** | Указание авторитетных DNS-серверов | `ns1.example.com` |

## ⚡ Оптимизации и особенности

- **Кэширование**: уменьшает нагрузку и ускоряет повторные запросы
- **Anycast**: один IP-адрес обслуживается множеством серверов по всему миру
- **DNS over HTTPS/TLS**: шифрование запросов для защиты приватности
- **Геолокация**: разные IP-адреса могут выдаваться в зависимости от региона пользователя

## 🛠️ Полезные команды для проверки

```bash
# Проверка домена (Linux/macOS)
nslookup google.com
dig google.com
host google.com

# Проверка конкретного типа записи
dig MX google.com
dig TXT google.com

# Windows
nslookup google.com
```

## 🔄 Пример полного пути запроса

```
Вы: "Открыть example.com"
   ↓
Браузер: кэш пуст → запрос ОС
   ↓
ОС: запрос к 8.8.8.8 (рекурсивный ресолвер)
   ↓
8.8.8.8: спрашивает корневой сервер "."
   ↓
Корневой: "TLD .com знает больше → 192.41.162.30"
   ↓
8.8.8.8: спрашивает TLD-сервер .com
   ↓
TLD: "Авторитетный сервер example.com → 93.184.216.34"
   ↓
8.8.8.8: запрашивает авторитетный сервер
   ↓
Авторитетный: "example.com = 93.184.216.34"
   ↓
Ответ возвращается вам, кэшируется, браузер подключается
```

## 🔐 Безопасность

- **DNSSEC**: добавляет цифровые подписи для проверки подлинности ответов
- **DoH/DoT**: шифрование трафика между клиентом и ресолвером
- **Фильтрация**: некоторые провайдеры блокируют вредоносные домены на уровне DNS
