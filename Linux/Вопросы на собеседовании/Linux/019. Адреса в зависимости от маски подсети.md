# Количество адресов в сети в зависимости от маски подсети

Вот таблица соответствия масок подсети (CIDR) и количества адресов в сети:

---

## 🧮 Формулы расчёта

```
Всего адресов = 2^(32 - CIDR)
Доступных хостов = 2^(32 - CIDR) - 2  (для масок /30 и крупнее)
```

**Пример для /24:**
- 32 − 24 = 8 бит под хосты
- 2⁸ = 256 всего адресов
- 256 − 2 = 254 доступных для устройств

---

## 🔑 Ключевые моменты

1. **Сетевой адрес** (первый в подсети) — идентифицирует саму сеть, не назначается хостам
2. **Широковещательный адрес** (последний в подсети) — для рассылки всем хостам сети
3. **Маска /32** — один конкретный хост (часто используется для маршрутов)
4. **Маска /31** — только для point-to-point соединений (оба адреса рабочие)
5. **Чем меньше число CIDR** → тем больше сеть → тем меньше подсетей можно создать

---

## 💡 Практический пример

```
Сеть: 192.168.1.0/26
- Маска: 255.255.255.192
- Всего адресов: 64
- Доступно хостам: 62
- Диапазон: 192.168.1.1 – 192.168.1.62
- Сетевой адрес: 192.168.1.0
- Broadcast: 192.168.1.63
```
