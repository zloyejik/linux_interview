# Зомби-процессы в Linux/Unix: что это, откуда берутся и в чем опасность

## 🔍 Что такое зомби-процесс?

**Зомби-процесс (zombie process)** — это процесс, который завершил своё выполнение, но всё ещё занимает запись в таблице процессов операционной системы. Такой процесс:

- ✅ Уже не выполняет код и не потребляет CPU/память
- ✅ Имеет статус `Z` (или `defunct`) в выводе `ps` или `top`
- ✅ Сохраняет только минимальную информацию: PID, статус завершения, данные об использовании ресурсов

```bash
$ ps aux | grep defunct
user  12345  0.0  0.0      0     0 pts/0    Z+   12:34   0:00 [my_process] <defunct>
```

## 🧬 Как появляется зомби?

Зомби возникает из-за особенностей механизма завершения процессов в Unix-подобных ОС:

```
1. Родительский процесс создаёт дочерний (fork)
2. Дочерний процесс завершает работу (exit)
3. ОС сохраняет статус завершения дочернего процесса
4. Родительский процесс ДОЛЖЕН вызвать wait()/waitpid(), чтобы "забрать" этот статус
5. Если родитель не вызывает wait() → дочерний процесс становится зомби
```

### Типичные причины:
- ❌ Программист забыл обработать завершение дочернего процесса
- ❌ Родительский процесс завис или работает в бесконечном цикле без обработки сигналов
- ❌ Неправильная работа с сигналами (например, игнорирование `SIGCHLD`)
- ❌ Долгоживущие сервисы, которые порождают много дочерних процессов без корректной уборки

## ⚠️ В чём опасность?

| Проблема | Описание |
|----------|----------|
| **Утечка PID** | Каждый зомби занимает PID. В системе ограниченное количество PID (обычно 32768+), их исчерпание помешает запускать новые процессы |
| **Переполнение таблицы процессов** | Ядро хранит информацию о зомби в памяти. Много зомби = нагрузка на ядро |
| **Сложность диагностики** | Зомби не видны как "нагрузочные" процессы, но могут маскировать реальные проблемы |
| **Признак багов в коде** | Наличие зомби часто указывает на ошибки в управлении процессами приложения |

> 💡 **Важно**: Сами по себе зомби не потребляют память или процессорное время, но их накопление — симптом более серьёзной проблемы.

## 🛠 Как найти и устранить?

### Поиск зомби:
```bash
# Найти все зомби-процессы
ps aux | awk '$8 == "Z"'

# Посчитать количество
ps aux | awk '$8 == "Z" {count++} END {print count}'

# Узнать родительский процесс зомби
ps -eo pid,ppid,stat,cmd | grep ' Z '
```

### Устранение:
1. **Идеальный способ**: Исправить код родительского процесса — добавить обработку `SIGCHLD` и вызов `waitpid()`:
   ```c
   signal(SIGCHLD, SIG_IGN); // или
   while (waitpid(-1, NULL, WNOHANG) > 0);
   ```

2. **Временное решение**: Перезапустить родительский процесс — тогда зомби будут "усыновлены" процессом `init` (PID 1), который автоматически выполнит `wait()`.

3. **Нельзя убить зомби через `kill`** — он уже мёртв, нужно "похоронить" его через родителя.

## 🎯 Профилактика

- Всегда обрабатывайте завершение дочерних процессов
- Используйте `sigaction()` для надёжной обработки сигналов
- Для долгих задач рассмотрите пулы процессов или очереди задач (например, через `systemd`, `supervisord`, `celery`)
- Мониторьте систему: настройте алерты на рост числа зомби
