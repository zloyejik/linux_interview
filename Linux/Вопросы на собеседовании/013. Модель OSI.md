# Модель OSI (Open Systems Interconnection)

**Модель OSI** — это эталонная модель взаимодействия открытых систем, разработанная Международной организацией по стандартизации (ISO) в 1984 году. Она описывает, как данные передаются по сети от одного устройства к другому, разделяя процесс на **7 уровней**.

---

## 📊 7 уровней модели OSI (снизу вверх)

| Уровень | Название (англ./рус.) | Единица данных | Примеры протоколов/технологий |
|---------|----------------------|----------------|------------------------------|
| **7** | Application / Прикладной | Данные | HTTP, HTTPS, FTP, SMTP, DNS, SSH |
| **6** | Presentation / Представления | Данные | SSL/TLS, JPEG, MPEG, ASCII, Unicode |
| **5** | Session / Сеансовый | Данные | NetBIOS, RPC, PPTP |
| **4** | Transport / Транспортный | Сегменты / Дейтаграммы | TCP, UDP, SCTP |
| **3** | Network / Сетевой | Пакеты | IP, ICMP, IPSec, routers |
| **2** | Data Link / Канальный | Кадры (фреймы) | Ethernet, Wi-Fi (802.11), MAC, switches |
| **1** | Physical / Физический | Биты | Кабели (UTP, оптоволокно), хабы, сигналы |

---

## 🔑 Ключевые принципы

### Инкапсуляция данных
При передаче данные проходят сверху вниз, и на каждом уровне добавляется свой заголовок:
```
[Данные] 
→ + заголовок L5-7 
→ + заголовок TCP/UDP (L4) 
→ + заголовок IP (L3) 
→ + заголовок Ethernet (L2) 
→ преобразование в биты (L1)
```

### Разделение ответственности
- Каждый уровень решает строго определённую задачу
- Уровни взаимодействуют только с соседними
- Возможность заменять реализации одного уровня без изменения других

---

## 🛠 Практическое применение

✅ **Диагностика сетей**: Помогает локализовать проблему (физический обрыв, ошибка маршрутизации, сбой приложения)  
✅ **Обучение**: База для понимания сетевых технологий  
✅ **Стандартизация**: Производители оборудования ориентируются на модель при разработке  
✅ **Сертификации**: Основа экзаменов CCNA, CompTIA Network+, и других

## ⚠️ Важное замечание

Модель OSI — **теоретическая**. В реальности используется более простая модель **TCP/IP** (4 уровня), но понимание OSI критически важно для:
- Глубокого анализа сетевых проблем
- Понимания работы протоколов
- Профессионального роста в IT-инфраструктуре
