## 🐳 Docker & Infrastructure
1. Контейнер падает с кодом 0/1, а в docker logs пусто. Что делать?

* Ответ: Процесс завершился слишком быстро или логи пишутся в файл, а не в stdout. Нужно запустить контейнер с переопределением точки входа: docker run --entrypoint sh -it <image>, чтобы проверить всё изнутри.

2. Как запустить контейнер не от root (User Namespaces)?

* Ответ: 1. Использовать инструкцию USER в Dockerfile. 2. Использовать режим Rootless Docker (демон работает от юзера). 3. Маппинг UID/GID: внутри контейнера ты root (0), снаружи — обычный юзер (1000).

3. Разница между Bridge и Host сетями?

* Ответ: Bridge: изоляция, свой IP, общение через виртуальный мост (NAT). Host: нет изоляции, контейнер использует порты хоста напрямую (макс. скорость, минимум защиты).

4. Как уменьшить размер образа?

* Ответ: Использовать Multi-stage builds (сборка в одном образе, запуск в другом), выбирать легкие базовые образы (Alpine), объединять команды RUN и чистить кэш пакетов (rm -rf /var/lib/apt/lists/*) в том же слое.

5. Почему docker stop ждет 10 секунд и убивает контейнер?

* Ответ: Процесс с PID 1 не пробрасывает сигнал SIGTERM. Решение: использовать JSON-форму в Dockerfile: ENTRYPOINT ["./app"] или флаг --init.

## 🛠 Troubleshooting & Database
6. Приложение долго авторизует пользователей. Куда смотреть?

* Ответ: 1. Ресурсы: Диск (I/O Wait), RAM, CPU. 2. БД: Медленные запросы, отсутствие индексов, блокировки таблиц. 3. Сеть: Проблемы с DNS или задержки до БД.

7. Как проверить, почему запрос к БД тормозит?

* Ответ: Использовать команду EXPLAIN ANALYZE. Она покажет, делает ли база Seq Scan (перебор всей таблицы) или Index Scan (быстрый поиск по индексу).

8. Решение проблемы "Too many connections" в БД?

* Ответ: Внедрить Connection Pooler (например, PgBouncer для Postgres) и проверить, закрывает ли приложение соединения.

## 📊 Monitoring (Zabbix)
9. Зачем нужен Zabbix Proxy?

* Ответ: Снятие нагрузки с сервера, мониторинг удаленных филиалов за NAT/Firewall, буферизация данных при разрыве связи.

10. Какие метрики говорят, что Zabbix не справляется?

* Ответ: Высокий NVPS, рост очереди (Queue), загрузка поллеров (Pollers) выше 75% и заполнение Value Cache.

## 🏗 Infrastructure as Code (Terraform & Ansible)
11. Что делать, если в облаке изменили ресурсы руками (Drift)?

* Ответ: Запустить terraform plan (увидеть разницу), затем terraform apply (вернуть как было) или обновить код .tf, чтобы он соответствовал облаку.

12. Как добавить в Terraform ресурс, созданный вручную?

* Ответ: Описать пустой блок ресурса в коде и выполнить terraform import <addr> <id>.

13. В чем разница между Terraform и Ansible?

* Ответ: Terraform — оркестрация (создание фундамента: сети, VM). Ansible — управление конфигурацией (настройка софта внутри VM, установка пакетов).
