

Это классическая ситуация в Linux. Причина заключается в том, что файл всё ещё открыт каким-то процессом.

В файловых системах Linux (например, ext4, xfs) удаление файла (rm) на самом деле не стирает данные сразу, если этот файл используется. Вот как это работает:

1. Механизм ссылок: У каждого файла есть счетчик ссылок (link count). Когда вы делаете rm, система просто уменьшает этот счетчик и удаляет имя файла из директории.
    
2. Удержание дескриптора: Если какой-то процесс (программа, демон, логгер) держит файл открытым (имеет открытый файловый дескриптор), ядро Linux не может освободить место на диске, пока этот процесс не закроет файл или не завершит работу. Данные остаются на диске, но доступ к ним по имени уже невозможен. В выводе команды df место будет занято, а в du — нет (так как du сканирует имена файлов в директориях).
    

Шаг 1: Найдите процессы, держащие удаленные файлы

Используйте команду lsof (list open files) с фильтром по слову "deleted":

lsof | grep deleted

Если lsof нет, можно использовать:

find /proc/*/fd -ls | grep deleted

Вы увидите вывод примерно такого вида:

COMMAND PID USER FD TYPE DEVICE SIZE/OFF NODE NAME

nginx 1234 root 4w REG 8,1 1048576 567 /var/log/nginx/access.log (deleted)

java 5678 app 2w REG 8,1 5242880 890 /tmp/large_temp_file (deleted)

Здесь видно имя процесса (COMMAND), его ID (PID) и путь к удаленному файлу.

Шаг 2: Освободите место

- Вариант А (Рекомендуемый): Перезапустить процесс.
    

Если это веб-сервер (nginx, apache), база данных или ваше приложение, лучше отправить ему сигнал перезагрузки или рестартовать службу. Это заставит процесс закрыть старые дескрипторы и открыть новые файлы.

*Пример для systemd
systemctl restart nginx

*Или отправить сигнал HUP (часто работает для перечитывания конфигов и логов)

kill -HUP <PID>

- Вариант Б (Радикальный): Убить процесс.
    

Если процесс завис или перезапуск невозможен, можно завершить его принудительно:

kill -9 <PID>

Осторожно: это может прервать выполнение важных задач

- Вариант В (Хак без перезапуска): Очистить файл через /proc.
    

Если вы не можете перезапустить процесс, но нужно срочно освободить место, можно обнулить содержимое файла напрямую через файловый дескриптор процесса:

> /proc/<PID>/fd/<FD>

(Замените <PID> на номер процесса, а <FD> на номер файлового дескриптора из вывода lsof).

После выполнения этих действий место на диске должно немедленно освободиться.
