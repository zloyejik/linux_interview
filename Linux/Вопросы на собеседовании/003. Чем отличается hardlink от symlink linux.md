 Linux hardlink (жесткая ссылка) и symlink (символическая ссылка, мягкая ссылка) — это два способа создать ссылку на файл, но работают они принципиально по-разному на уровне файловой системы.

 Давайте разберем это подробно, заглянув «под капот» файловой системы (на примере распространенной ext4).

Чтобы понять разницу, нужно знать три ключевых понятия:
1.  **Inode (инода):** Структура данных, где хранится метаинформация о файле (права доступа, владелец, время создания) и указатели на то, где физически лежат данные.
2.  **Data Blocks (блоки данных):** Место на диске, где хранится само содержимое файла (текст, код, картинка).
3.  **Directory Entry (запись в каталоге):** Строка в специальной файле-каталоге, которая связывает **Имя файла** с номером **Inode**.

---

### 1. Hardlink: «Не занимает места (только запись в каталоге)»

Когда вы создаете жесткую ссылку, файловая система **не создает новый файл**.

*   **Что происходит:** Система просто добавляет новую строку в таблицу каталога (папки).
*   **Структура записи:** `[Новое Имя] -> [Номер Inode оригинала]`.
*   **Inode:** Не создается новый. Используется тот же номер, что и у оригинала.
*   **Данные:** Не копируются. Указатель ведет на те же блоки данных.

**Почему это почти 0 байт?**
Запись в каталоге занимает очень мало места (обычно несколько десятков байт), потому что хранит только имя и номер иноды. Это служебная информация самой папки.
Для системы жесткая ссылка неотличима от оригинала. Если вы посмотрите размер файла через `ls -l`, вы увидите размер **исходных данных**, а не размер ссылки.

> **Аналогия:** Представьте, что файл — это человек с паспортом (Inode).
> Hardlink — это просто запись в телефонной книге: «Иван Иванов».
> Если вы добавите вторую запись «Ваня Иванов» на тот же номер телефона, вы не создали нового человека. Вы просто потратили немного чернил и бумаги на вторую строчку в записной книжке.

---

### 2. Symlink: «Занимает место под хранение пути к файлу»

Символическая ссылка — это **полноценный отдельный файл**.

*   **Что происходит:** Система создает **новый Inode** для самой ссылки.
*   **Данные:** В блоки данных этого нового файла записывается **текстовая строка** — путь к целевому файлу (например, `/home/user/documents/report.txt`).
*   **Размер:** Размер файла-ссылки равен длине этой строки пути.

**Почему это занимает место?**
1.  **Расход иноды:** Под ссылку резервируется структура inode (метаданные самой ссылки).
2.  **Расход места под путь:** Если путь длинный, файл ссылки будет «тяжелее».
    *   Ссылка на `file.txt` займет ~10 байт.
    *   Ссылка на `/very/long/path/to/some/deeply/nested/file.txt` займет ~50 байт.

> **Аналогия:** Symlink — это бумажная записка, лежащая на столе.
> На записке написано: «Иди в кабинет 101».
> Сама записка — это отдельный объект. Она занимает место на столе (диске). Если вы напишете очень длинный адрес, вам понадобится лист бумаги большего размера.

---

### 3. Практический пример (как это увидеть)

Давайте создадим оба типа ссылок и посмотрим на их размер и иноды.

```bash
# 1. Создаем исходный файл (100 байт данных)
dd if=/dev/zero of=original.txt bs=1 count=100

# 2. Создаем жесткую ссылку
ln original.txt hard_link.txt

# 3. Создаем символьную ссылку
ln -s original.txt soft_link.txt

# 4. Смотрим детали (флаг -i показывает номер inode, -l показывает размер)
ls -li
```

**Пример вывода:**
```text
123456 -rw-r--r-- 2 user user 100 янв 1 12:00 original.txt
123456 -rw-r--r-- 2 user user 100 янв 1 12:00 hard_link.txt
789012 lrwxrwxrwx 1 user user  12 янв 1 12:00 soft_link.txt -> original.txt
```

**Разбор вывода:**
1.  **Inode (первая колонка):**
    *   `original.txt` и `hard_link.txt` имеют одинаковый номер (`123456`). Это один и тот же файл.
    *   `soft_link.txt` имеет свой номер (`789012`). Это отдельный объект.
2.  **Размер (пятая колонка):**
    *   У `original.txt` и `hard_link.txt` размер **100 байт** (размер реальных данных).
    *   У `soft_link.txt` размер **12 байт**. Почему 12? Потому что слово `original.txt` состоит из 12 символов. Ссылка хранит именно этот текст.

### Итог
*   **Hardlink:** Не дублирует данные и не создает новый объект файла. Тратит лишь крошечное место на имя в списке папки.
*   **Symlink:** Создает новый файл-объект, который хранит в себе текст пути. Чем длиннее путь, тем больше места занимает ссылка.
