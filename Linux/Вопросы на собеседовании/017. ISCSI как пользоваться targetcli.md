`targetcli` — это интерактивная оболочка для управления подсистемой **LIO (Linux-IO Target)** в ядре Linux. С её помощью можно создавать iSCSI, FCoE, SRP и другие сетевые хранилища, превращая обычный Linux-сервер в полноценный SAN/NAS.

Ниже приведён практический гайд по использованию `targetcli` на примере настройки **iSCSI-таргета**.

---
## 🔹 1. Установка
```bash
# Debian/Ubuntu
sudo apt update && sudo apt install targetcli-fs

# RHEL/Rocky/AlmaLinux/Fedora
sudo dnf install targetcli
```
Пакет также тянет зависимости ядра (`target-core`, `rtslib-fb`).

---
## 🔹 2. Запуск и интерфейс
```bash
sudo targetcli
```
Оболочка выглядит как дерево каталогов. Навигация и команды похожи на bash:
- `ls` / `cd` — просмотр и переход
- `create` / `delete` — создание/удаление объектов
- `set` / `get` — изменение параметров
- `help` — справка по текущему контексту
- `exit` — выход (автоматически сохраняет конфигурацию)

---
## 🔹 3. Основные понятия
| Объект | Описание |
|--------|----------|
| `backstores` | Источники хранилища: блочные устройства, файлы, RAM-диски |
| `iscsi` (или `fc`, `srp`) | Протокол таргета |
| `tpg1` | Target Port Group: содержит порталы, LUN и ACL |
| `portals` | IP-адрес и порт (по умолчанию `0.0.0.0:3260`) |
| `luns` | Логические устройства, привязанные к backstore |
| `acls` | Контроль доступа по IQN инициатора |

---
## 🔹 4. Пошаговый пример: iSCSI Target
Предположим, вы хотите расшарить диск `/dev/sdb` по iSCSI.

### ① Создаём backstore
```bash
/> cd /backstores/block
/backstores/block> create disk0 /dev/vg_iscsi/lv_iscsi
```
*(Можно использовать `fileio`, `ramdisk` и т.д.)*
==Важно: лучше не подключать диски вида /dev/sdX, т.к. при перезагрузке диски могут менять названия sdX. Для этого диски подключают в виде LVM или по UUID==

### ② Создаём iSCSI-таргет
```bash
/> cd /iscsi
/iscsi> create iqn.2026-04.local:storage01
```
Автоматически создаётся `tpg1`. IQN должен соответствовать формату:  
`iqn.YYYY-MM.reversed.domain:label`

### ③ Настраиваем TPG
```bash
/iscsi> cd iqn.2026-04.local:storage01/tpg1
```

### ④ Добавляем портал (IP:порт)
```bash
/iscsi/.../tpg1> portals/ create 0.0.0.0 3260
```
*(Используйте конкретный IP сервера вместо `0.0.0.0` для безопасности)*

### ⑤ Привязываем LUN
```bash
/iscsi/.../tpg1> luns/ create /backstores/block/disk0
```

### ⑥ Разрешаем доступ (ACL)
Узнайте IQN клиента (на клиенте: `cat /etc/iscsi/initiatorname.iscsi` или `iscsi-iname`), затем:
```bash
/iscsi/.../tpg1> acls/ create iqn.2026-04.local:client01
```
*(Без ACL таргет будет отклонять подключения)*

### ⑦ Сохраняем и выходим
```bash
/iscsi/.../tpg1> cd /
/> saveconfig
/> exit
```
Конфигурация сохраняется в `/etc/target/saveconfig.json`.

---
## 🔹 5. Включение и автозагрузка
```bash
sudo systemctl enable --now target
sudo systemctl status target
```
Сервис `target` загружает сохранённую конфигурацию при старте.

---
## 🔹 6. Полезные команды
| Команда | Описание |
|--------|----------|
| `ls` | Дерево всех объектов |
| `status` | Проверка статуса LIO |
| `clearconfig confirm=True` | Полный сброс конфигурации |
| `get attribute authentication` | Проверка настроек аутентификации |
| `set attribute ...` | Изменение параметров (например, `demo_mode_write_protect=0`) |

---
## 🔹 7. Важные замечания
⚠️ **Брандмауэр**: Откройте порт `3260/tcp`
```bash
sudo ufw allow 3260/tcp   # или firewall-cmd --add-port=3260/tcp --permanent
```

⚠️ **SELinux/AppArmor**: На системах с SELinux может потребоваться:
```bash
sudo setsebool -P targetd_use_nfs 1  # если используется NFS backstore
sudo restorecon -Rv /etc/target/
```

⚠️ **Безопасность**: 
- Всегда указывайте конкретный IP в `portals/`, а не `0.0.0.0`
- Используйте CHAP-аутентификацию для production:
  ```bash
  /> set auth userid=alice password=secret
  /> set auth mutual_userid=bob mutual_password=secret2
  ```

⚠️ **Инициатор на клиенте** (Linux):
```bash
sudo apt install open-iscsi
sudo iscsiadm -m discovery -t st -p <IP_сервера>
sudo iscsiadm -m node -l
```
