# Что такое LVM (Logical Volume Manager)

**LVM (Logical Volume Manager)** — это система управления дисковым пространством в Linux, которая добавляет уровень абстракции между физическими дисками и файловыми системами [[5]]. Она позволяет объединять разные области одного или нескольких жёстких дисков в единые логические тома, предоставляя гибкость в управлении хранилищем данных.

---

## 🏗️ Архитектура LVM

```
Физические диски/разделы
        ↓
Физические тома (PV — Physical Volume)
        ↓
Группа томов (VG — Volume Group) ← объединяет несколько PV
        ↓
Логические тома (LV — Logical Volume) ← аналог обычных разделов
        ↓
Файловая система (ext4, XFS, Btrfs и др.)
```

### Ключевые компоненты [[2]][[11]]:

| Компонент | Описание |
|-----------|----------|
| **PV (Physical Volume)** | Физический раздел или целый диск, подготовленный для использования LVM |
| **VG (Volume Group)** | Группа физических томов, представляющая единый пул дискового пространства |
| **LV (Logical Volume)** | Логический раздел, создаваемый внутри группы томов; виден системе как обычное блочное устройство |
| **PE/LE (Physical/Logical Extent)** | Блоки фиксированного размера (обычно 4–32 МБ), на которые разбиваются PV и LV для управления пространством |

---

## ✅ Основные преимущества LVM

1. **Гибкое изменение размеров** — можно увеличивать (а иногда и уменьшать) логические тома «на лету», без перезагрузки системы [[7]][[12]].
2. **Объединение дисков** — одна группа томов может включать диски разного размера и типа [[2]].
3. **Снапшоты (snapshots)** — мгновенные снимки состояния тома для резервного копирования или тестирования [[11]].
4. **Миграция данных** — возможность переносить данные между физическими дисками без простоя (`pvmove`) [[11]].
5. **Зеркалирование** — поддержка RAID-1-подобной конфигурации для повышения отказоустойчивости [[2]].

## 🔧 Базовые команды LVM

```bash
# Создание физического тома
pvcreate /dev/sdb1

# Создание группы томов
vgcreate vg_data /dev/sdb1 /dev/sdc1

# Создание логического тома
lvcreate -n lv_home -L 50G vg_data

# Форматирование и монтирование
mkfs.ext4 /dev/vg_data/lv_home
mount /dev/vg_data/lv_home /home

# Расширение тома «на лету»
lvresize -L +10G vg_data/lv_home
resize2fs /dev/vg_data/lv_home  # для ext4

# Создание снапшота
lvcreate -s -n lv_home_snap -L 5G vg_data/lv_home
```

---

## ⚠️ Важные ограничения

- Уменьшение размера тома возможно **только** для файловых систем, поддерживающих эту операцию (например, ext4), и требует предварительной проверки целостности данных [[5]].
- Снапшоты занимают место в группе томов и не заменяют полноценное резервное копирование на отдельный носитель [[11]].
- LVM **не обеспечивает отказоустойчивость** сам по себе — для этого его обычно комбинируют с RAID [[2]].

---

## 💡 Когда использовать LVM?

| Сценарий | Почему LVM подходит |
|----------|---------------------|
| Сервер с растущими данными | Легко добавить диск и расширить том без переустановки |
| Тестирование обновлений | Снапшот позволяет быстро откатить изменения |
| Виртуализация | Гибкое распределение пространства между ВМ |
| Домашний NAS | Объединение дисков разного размера в единое хранилище |
